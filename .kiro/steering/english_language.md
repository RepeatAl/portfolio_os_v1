# English-Only Documentation Policy

## inclusion: always

**Status**: MANDATORY | **Owner**: CTO | **Scope**: all-artifacts

All files created by Kiro MUST be written in English. This includes: specs, reports, governance docs, architecture decisions, code comments, commit messages, README files.

Allowed exceptions:
- i18n translation files (`public/locales/de/*.json`)
- User-facing copy in translation files
- Legal documents requiring German (Impressum, Datenschutz)
- Quoting German requirements from external sources

When modifying existing files: new content MUST be in English. Existing German content MAY remain (legacy). Full rewrites MUST be in English.
