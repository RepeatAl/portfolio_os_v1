# Report Output Registration Guide

## Overview

This guide documents the registration of report output artifacts (REPORT_OUT type) in the domainization system. Report outputs are generated text files that contain human-readable portfolio analysis, briefings, and reports for portfolio managers.

## Purpose

Report outputs represent the final stage of the Portfolio OS reasoning chain:
```
SIGNALS → SEMANTICS → REASONING → REPORT
```

These artifacts are:
- Generated by report engines
- Written in human-readable format
- Consumed by portfolio managers
- Updated daily or on-demand
- Owned by the REPORT domain

## Artifact Type: REPORT_OUT

### Lifecycle States

Report outputs follow this lifecycle:
- **generated**: Report has been created by an engine
- **current**: Report is the latest version
- **archived**: Report has been superseded by a newer version

### Metadata Requirements

All REPORT_OUT artifacts must include:

**Required Fields:**
- `artifact_id`: Unique identifier (e.g., `portfolio_report_txt`)
- `file_path`: Relative path from repo root (e.g., `portfolio_report.txt`)
- `primary_domain`: Must be `REPORT`
- `artifact_type`: Must be `REPORT_OUT`
- `lifecycle_status`: One of: generated, current, archived
- `created_date`: Creation date (YYYY-MM-DD)
- `last_modified`: Last modification date (YYYY-MM-DD)
- `owner_role`: Typically "Report generation system"
- `ssot_relationship`: Must be `none` (reports are generated, not canonical)
- `allowed_writers`: Must include `REPORT` domain
- `allowed_readers`: Typically `ALL` for public access

**Optional but Recommended:**
- `dependencies`: List of artifact IDs (engines and SSOT docs) that generate this report
- `description`: Human-readable description of report content
- `tags`: Categorization tags (e.g., report, briefing, daily)

## Authority Chain Compliance

All report outputs comply with the authority chain:

```
SIGNALS → SEMANTICS → REASONING → REPORT
```

- **SIGNALS domain**: Generates raw signals (allocation, regime, attribution engines)
- **SEMANTICS domain**: Interprets signals into semantic states
- **REASONING domain**: Creates reasoning conclusions
- **REPORT domain**: Transforms reasoning into human-readable reports

Report outputs are the final stage and:
- ✅ Only write human-readable text
- ✅ Do not create business logic
- ✅ Read from reasoning outputs
- ✅ Reference upstream SSOT documents

## Usage Examples

### Query Report Outputs by Domain

```python
from artifact_registry import ArtifactRegistry

registry = ArtifactRegistry()
registry.load()

# Get all REPORT domain artifacts
report_artifacts = registry.list_artifacts_by_domain('REPORT')

# Filter for REPORT_OUT type
report_outputs = [a for a in report_artifacts if a.artifact_type == 'REPORT_OUT']

print(f'Found {len(report_outputs)} report outputs')
```

### Check Report Dependencies

```python
from artifact_registry import ArtifactRegistry

registry = ArtifactRegistry()
registry.load()

# Get portfolio report
portfolio_report = registry.get_artifact('portfolio_report_txt')

print(f'Portfolio Report Dependencies:')
for dep_id in portfolio_report.dependencies:
    dep = registry.get_artifact(dep_id)
    if dep:
        print(f'  - {dep.artifact_id} ({dep.artifact_type})')
```

## Maintenance

### Adding New Report Outputs

When adding a new report output:

1. **Create the report file** (e.g., `new_analysis_briefing.txt`)

2. **Register in artifact_registry.yaml**:
```yaml
- artifact_id: new_analysis_briefing_txt
  file_path: new_analysis_briefing.txt
  primary_domain: REPORT
  artifact_type: REPORT_OUT
  lifecycle_status: current
  created_date: "YYYY-MM-DD"
  last_modified: "YYYY-MM-DD"
  owner_role: Report generation system
  ssot_relationship: none
  allowed_writers:
    - REPORT
  allowed_readers:
    - ALL
  dependencies:
    - generating_engine_py
    - report_reasoning_system_md
  description: "Description of the new report"
  tags:
    - report
    - briefing
    - analysis
```

3. **Validate registration**:
```bash
python -c "
from artifact_registry import ArtifactRegistry
registry = ArtifactRegistry()
registry.load()
artifact = registry.get_artifact('new_analysis_briefing_txt')
print(f'✓ Registered: {artifact.file_path}')
"
```

### Archiving Old Reports

When a report is superseded:

1. Update lifecycle_status to `archived`
2. Update last_modified date
3. Optionally move file to `history/` directory
4. Update file_path in registry if moved

### Updating Report Dependencies

When report generation logic changes:

1. Update the `dependencies` list
2. Update `last_modified` date
3. Ensure all referenced artifacts exist in registry
