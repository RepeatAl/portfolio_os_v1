# PORTFOLIO OS — CORRELATION & DEPENDENCY FRAMEWORK
Version: v1
Status: Canonical Dependency SSOT

---

# PURPOSE

This document defines how Portfolio OS evaluates:

- correlations
- dependency chains
- thematic overlap
- hidden concentration
- structural exposure amplification

Portfolio OS does NOT interpret assets in isolation.

The system evaluates:

how assets behave together.

The framework exists to answer:

"What hidden structures actually drive the portfolio?"

---

# CORE PRINCIPLE

Diversification is not position count.

A portfolio may contain many assets
while remaining structurally dependent on:

- one narrative
- one liquidity condition
- one macro factor
- one growth regime
- one thematic driver

Portfolio OS evaluates structural dependency,
not cosmetic diversification.

---

# CORRELATION CHAIN

Portfolio OS follows this structure:

Portfolio State
→ Exposure Mapping
→ Correlation Detection
→ Dependency Analysis
→ Semantic Interpretation
→ PM Reasoning
→ Structural Risk Assessment

Dependency meaning emerges through relationships.

---

# FRAMEWORK OBJECTIVE

The framework should answer:

- Which assets behave similarly?
- Which themes overlap?
- Which narratives dominate?
- Which exposures amplify each other?
- Which hidden fragilities exist?
- Which diversification is real?
- Which diversification is cosmetic?

The goal is structural transparency.

# DEPENDENCY STATES

Portfolio OS recognizes multiple dependency states.

---

# dependency_balanced

Meaning:

- diversified structural drivers
- manageable overlap
- broad exposure support

Implications:

Portfolio resilience remains relatively stable.

---

# dependency_elevated

Meaning:

- several positions depend on similar conditions
- thematic overlap increasing
- concentration chains forming

Implications:

Portfolio sensitivity rises under stress.

---

# dependency_fragile

Meaning:

- portfolio heavily depends on limited drivers
- hidden overlap dominates
- downside amplification risk rising

Implications:

Single structural weakness may affect multiple positions simultaneously.

---

# dependency_extreme

Meaning:

- severe thematic concentration
- highly correlated structure
- limited structural diversification

Implications:

Portfolio resilience becomes highly fragile under adverse conditions.

---

# DEPENDENCY INTERPRETATION RULE

The system must ALWAYS explain:

- why dependency exists
- which positions overlap
- which narratives dominate
- which hidden structures amplify fragility

Bad Example:

"Correlation elevated."

Good Example:

"Several portfolio positions currently depend
on continued AI infrastructure strength,
which increases sensitivity if AI spending weakens."

---

# COSMETIC DIVERSIFICATION RULE

Portfolio OS explicitly separates:

Cosmetic Diversification
from
Structural Diversification.

Example:

A portfolio may contain:

- NVIDIA
- ASML
- Microsoft
- AMD

while still remaining heavily dependent on:

AI-related growth leadership.

The system must detect hidden clustering.

---

# DEPENDENCY VS QUALITY

High-quality companies may still create dependency concentration.

Example:

Microsoft and NVIDIA may both remain strong businesses.

However:

combined exposure may still create:

- narrative overlap
- liquidity overlap
- macro overlap

The framework evaluates structural interaction,
not isolated company quality.

---
