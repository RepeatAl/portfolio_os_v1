# PORTFOLIO OS — SCORING METHODOLOGY FRAMEWORK
Version: v1
Status: Canonical Scoring SSOT

---

# PURPOSE

This document defines how Portfolio OS calculates scores.

Portfolio OS scores are NOT:

- prediction scores
- performance guarantees
- emotional rankings
- hype indicators

Scores represent:

- structural quality
- resilience
- alignment
- stability
- portfolio coherence
- deployment quality
- reasoning integrity

The purpose of scoring is to compress complex portfolio structure
into explainable interpretive signals.

---

# CORE PRINCIPLE

Scores summarize structure.

Scores do NOT replace reasoning.

Portfolio OS must NEVER allow scores to become:

- magical numbers
- black-box ratings
- authority simulation
- unexplained conclusions

Every score must remain:

- explainable
- traceable
- decomposable
- structurally grounded

---

# SCORING CHAIN

Portfolio OS follows this structure:

Raw Signals
→ Structural Metrics
→ Semantic States
→ Weighted Interpretation
→ Score Calculation
→ PM Reasoning

Scores emerge from semantic structure,
not isolated metrics.

---

# SCORING OBJECTIVE

The scoring framework should answer:

- How structurally healthy is the portfolio?
- How resilient is current positioning?
- How aligned are signals?
- How stable is deployment?
- How fragile are dependencies?
- How coherent is the regime structure?

Scores exist to support understanding,
not to replace it.

---

# SCORING TYPES

Portfolio OS supports multiple score classes.

---

# TYPE 1 — PORTFOLIO HEALTH SCORE

Purpose:
Measure structural portfolio resilience.

Evaluates:

- concentration
- diversification
- dependency overlap
- scenario fragility
- deployment balance

Questions answered:

- How structurally resilient is the portfolio?
- How balanced is exposure?

---

# TYPE 2 — CONFIDENCE SCORE

Purpose:
Measure reasoning stability.

Evaluates:

- signal alignment
- semantic consistency
- cross-engine confirmation
- scenario coherence

Questions answered:

- How coherent is the current interpretation?
- How much ambiguity exists?

---

# TYPE 3 — DEPLOYMENT SCORE

Purpose:
Measure capital positioning quality.

Evaluates:

- flexibility
- optionality
- exposure pacing
- deployment concentration

Questions answered:

- How adaptive is capital structure?
- How rigid is positioning?

---

# TYPE 4 — REGIME ALIGNMENT SCORE

Purpose:
Measure compatibility between portfolio and environment.

Evaluates:

- liquidity alignment
- breadth compatibility
- volatility sensitivity
- macro consistency

Questions answered:

- Does the market environment support the portfolio?

---

# TYPE 5 — OPPORTUNITY SCORE

Purpose:
Measure structural opportunity quality.

Evaluates:

- diversification contribution
- dependency reduction
- resilience improvement
- deployment efficiency

Questions answered:

- Does this improve portfolio structure?

# SCORE NORMALIZATION RULE

All scores should normalize into:

0–100 scale.

Examples:

- 0–20 → Structurally Critical
- 21–40 → Fragile
- 41–60 → Moderate
- 61–80 → Stable
- 81–100 → Strong Structural Quality

The system must preserve:

- consistency
- interpretability
- comparability

---

# WEIGHTING PRINCIPLE

Not all dimensions carry equal importance.

Portfolio OS prioritizes:

1. Structural fragility
2. Diversification quality
3. Dependency overlap
4. Liquidity compatibility
5. Scenario survivability
6. Deployment quality
7. Tactical participation

Structural stability dominates tactical strength.

---

# EXAMPLE SCORING MODEL

Example:

Portfolio Health Score

Inputs:

- concentration risk
- dependency overlap
- diversification quality
- scenario resilience
- deployment balance

Weighted output:

- 78 → structurally stable
- 42 → elevated fragility
- 25 → critical dependency structure

Scores summarize structure,
not emotional conviction.

---

# SCORE INTERPRETATION RULE

The system must ALWAYS explain:

- why a score exists
- what increased the score
- what weakened the score
- what structural conditions dominate

Bad Example:

"Portfolio Score: 74"

Good Example:

"The portfolio maintains balanced deployment
and manageable diversification quality,
though AI-related dependency remains elevated."

---

# SCORE VS PREDICTION

Portfolio OS explicitly separates:

Structural Score
from
Future Prediction.

Example:

A portfolio may score highly because:

- diversification healthy
- deployment balanced
- resilience strong

without guaranteeing:

future outperformance.

Scores evaluate structure,
not certainty.

---

# SCORE VS PERFORMANCE

Strong performance does NOT automatically create strong scores.

Example:

A highly concentrated AI portfolio may:

- strongly outperform
while simultaneously:
- weakening structural resilience

Portfolio OS evaluates:

portfolio structure,
not emotional performance satisfaction.

---

# SCORE CONFLICT RULE

Conflicting signals reduce score confidence.

Example:

- supportive liquidity
BUT
- extreme concentration
AND
- narrow participation

This may produce:

mixed structural scoring.

The system must explain contradictions openly.

---

# SEMANTIC SCORING MODEL

Scores emerge from semantic interaction.

Examples:

- ai_dependency_high
- concentration_risk_elevated
- participation_narrow

combined together may reduce:

Portfolio Health Score

Meaning emerges through interaction.

---

# SCORE STABILITY RULE

Scores should avoid hypersensitive fluctuations.

Portfolio OS should prefer:

- structural persistence
over
- noisy short-term oscillation

Scores must reflect:

meaningful structural change.

---

# SCORE TRACEABILITY RULE

Every score must remain traceable to:

- source signals
- semantic states
- weighting logic
- normalization method
- engine outputs

No score may become a black box.

---

# HUMAN READABILITY RULE

Score explanations must remain understandable.

The system must explain:

- what the score measures
- what structurally matters
- what improves resilience
- what weakens structure
- what creates fragility

The system must avoid:

- quant jargon
- pseudo-intelligent wording
- unexplained ratings
- institutional opacity

---

# REPORT RELATIONSHIP

Reports explain score meaning.

Reports should answer:

- why scores changed
- what structurally improved
- what weakened resilience
- what dependencies dominate
- what fragilities emerged

Reports interpret scores,
not merely display them.
